Spiralcraft

spiralcraft-core module

The spiralcraft-core module provides a set of subsystems which support the integration of application components. Add-on modules use mechanisms in the spiralcraft-core module to expose data and functionality to application components, system management utilities, development tools and user interfaces..

Functional Groups

Runtime control

Packages associated with tracking and running code in a managed execution environment.
  • spiralcraft.loader
    Contains mechanisms which control the loading, unloading and sharing of code modules via the use of a library of versioning information supplied by the individual modules. This package used in conjunction with the spiralcraft-main module permits muliple versions of a module to simultaneously run in the same VM under different application contexts.
  • spiralcraft.exec
    Manages the instantiation and activation of application contexts, which are relatively independent, cohesive collections of  functionality and resources.
  • spiralcraft.command
    Command line interface provides access to all application and foundation functionality via a scriptable, text based system.

Application foundation 

Packages which implement framework methodologies that underpin Spiralcraft applications..
  • spiralcraft.builder
    A system for composing applications by defining 'Assemblies' using an XML dialect:
    • Separates declarative information which specifies composition and configuration from Java code which specifies component behavior.
    • Assumes the responsibilities of component instantiation, configuration, and interconnection to increase reusability and decrease complexity of Java components. 
    • Provides an inheritance mechanism to permit multi-level customization of Assemblies. 
    • Provides local-singleton and expression-language based mechanisms to simplify the interconnection of components, the resolution of configuration parameters, and the seamless integration of add-on systems.
    • Forms a stable namespace hierarchy which makes application components reachable for purposes of design, configuration and monitoring.
    • Implements persistence that is tolerant to code updates by using information specified in Assemblies.
  • spiralcraft.lang
    The implementation of the expression language system used throughout the Spiralcraft architecture.. Provides the ability to expose and access dynamic data models at runtime using an intuitive Java-like path expression syntax. Provides an API to allow developers to use expressions against existing object models, and an SPI for developers to extend the reach of the expression language into alternative object models.
  • spiralcraft.service
    • Foundation of an application architecture where the application is composed of various pluggable, interdependent Services with well defined life cycles which expose a wide variety of interfaces for use by user-driven compoenents.
  • spiralcraft.ui
    A User Interface abstraction which provides a common set of interfaces and abstract classes for implementing the Controller role within the Hierarchical Model/View/Controller design pattern. The UI package is extended by toolkit specific View implementations and business logic specific Model implementations, enabling a many-to-many relationship between them in order to maximize code re-use.
  • spiralcraft.reference
    A set of interfaces which allows different object stores (filesystems, databases, directories, web sites) to reference each other's objects within a well defined group of services. Assists the integration of file systems, archives, databases and other object stores by supporting the import, export and storage of object references using locally unique identifiers (LUIDs) generated from the native keying mechanism of the store.
  • spiralcraft.telemetry
    (Planned) Interface which provides a standard means for components to report operational statistics while ensuring minimum impact on the runtime system profile. System is composed of a tree of 'Meters', each of which contains 'Logs' for capuring events and 'Registers' for capturing statistics.
  • spiralcraft.task
    A mechanism for invoking granular units of functionality and reporting detailed progress to a user interface or agent.
  • spiralcraft.tuple
    A system for representing and manipulating relational data.
Input/Output
  • spiralcraft.xml
    General purpose XML parsing and processing utilities for applications which manipulate XML fragments or provide interfaces based on subsets or variants of XML.
  • spiralcraft.sax
    XML utilities based on SAX (simple API for XML) to facilitate the reading and writing of XML data. Contains a parse tree to help cache SAX event sequences.
  • spiralcraft.stream
    Provides an interface for applications to manipulate byte level data from arbitrary sources using URIs. Includes support for relative paths, containership, and capabilities inquiry. Most components which need to open streams and traverse resource container hierarchies can be implemented against these interfaces for transparent support of present and future 'stream resources', for example files, jar archives, web servers, cvs repositories, and devices. Also contains utilities to help with various aspects of programming streams.
    • spiralcraft.stream.batch
      Batch oriented utilities to perform container hierarchy traversal based operations such as searches and batch transformations (eg. global search and replace).
  • spiralcraft.prefs
    Tools based on the java.util.prefs API to support the loading and storing of user and installation specific configuration data.
  • spiralcraft.text
    • Tools and interfaces to manipulate and filter character based data.

Management

  • spiralcraft.registry
    Facilitates management by providing a unified means of bidirectional access from application object models to management tools for purposes of configuration, monitoring and control.
  • spiralcraft.log
    Utilities which integrates the java logging package with the spiralcraft.registry package to facilitate the inclusion and fine grained control of logging functionality into most application components.

Utility

Packages which facilitate common programming tasks
  • spiralcraft.util
    Utilities to assist common programming tasks. Contains tools to manipulate Strings, Arrays, and command line arguments and perform common data conversions. 
  • spiralcraft.beans
    Tools to manipulate and augment the java.beans API. Provides support for caching BeanInfo results.
  • spiralcraft.pool
    An abstracted implementation of a pool, defined as a collection of resources which grows and shrinks as needed to provide low latency access to scarce resources.
  • spiralcraft.time
    Tools, such as a Clock and a Schedule, for implementing time-based functionality.
